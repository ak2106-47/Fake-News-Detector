# Fake Content Detection for Social Media

**Author: Apoorva Kumar**

A machine learning web application that detects fake news and misinformation by combining a trained XGBoost classifier with live fact-checking APIs.

---

## Demo

![App Screenshot](screenshots/app.png)

Try entering a news headline to get an instant prediction along with supporting evidence from Google's Fact Check database.

---

## Features

- **ML-based text classification** — XGBoost model trained on 44,000+ real and fake news articles
- **Google Fact Check integration** — surfaces related claims and verdicts from verified fact-checkers
- **ClaimBuster integration** — scores how check-worthy a claim is
- **User feedback loop** — collects corrections via SQLite to support future retraining
- **Flask web interface** — clean, responsive UI for submitting and reviewing claims

---

## Model Performance

| Metric | Score |
|---|---|
| Accuracy | 93.89% |
| Precision | 91.80% |
| Recall | 97.94% |
| F1-Score | 94.77% |

Trained on the [ISOT Fake News Dataset](https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset) using TF-IDF vectorization with trigrams and an XGBoost classifier.

## How It Works

1. User submits a news claim via the web interface
2. Text is cleaned and stemmed using NLTK
3. TF-IDF vectorizer transforms the text into features
4. XGBoost model predicts **Genuine** or **Fake**
5. Google Fact Check API returns related verified claims
6. ClaimBuster scores how check-worthy the claim is
7. User can submit feedback to flag incorrect predictions

## Run Locally

### Prerequisites
- Python 3.9 or higher
- pip3 (comes with Python)
- A free [Kaggle account](https://www.kaggle.com) to download the dataset
- A free [Google Cloud account](https://console.cloud.google.com) to get a Fact Check API key
- A free [ClaimBuster account](https://idir.uta.edu/claimbuster) to get an API key

---

**1. Clone the repo**
```bash
git clone <your-repo-url>
cd Fake-Content-Detection-Social-Media
```

**2. Install dependencies**
```bash
pip3 install -r model/requirements.txt
```

> On Windows, use `pip` instead of `pip3`

**3. Set up API keys**

Copy `.env.example` to `.env`:
```bash
cp model/.env.example model/.env
```

Open `model/.env` and fill in your API keys:
```
FACT_CHECK_API_KEY=your_google_fact_check_api_key_here
CLAIMBUSTER_API_KEY=your_claimbuster_api_key_here
```

> The app will still run without API keys — the Google Fact Check and ClaimBuster sections will just show errors, but the ML prediction will still work.

**4. Download the dataset**

- Go to [https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset](https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset)
- Sign in to Kaggle (free account required)
- Click **Download** and unzip the file
- Place `Fake.csv` and `True.csv` into the `Datasets/` folder

**5. Build the model**

This trains the XGBoost model and saves `model.pkl` and `vectorizer.pkl`. It will take **5–15 minutes** depending on your machine.

```bash
cd model
python3 build_model.py
```

> On Windows, use `python` instead of `python3`

**6. Start the app**
```bash
python3 app.py
```

**7. Open in your browser**
```
http://localhost:5000
```

## Limitations & Future Work

- The current model uses TF-IDF + XGBoost, which is effective but does not understand context the way transformer models do. A future version could fine-tune a **BERT** or **RoBERTa** model for improved accuracy on short or ambiguous claims.
- The feedback collected in SQLite could be used to periodically retrain the model via `retrain_model.py`.
