# Fake Content Detection for Social Media

**Author: Apoorva**

A machine learning web application that detects fake news and misinformation by combining a trained XGBoost classifier with live fact-checking APIs.

---

![App Screenshot](screenshots/app.png)

## Features

- **ML-based text classification** — XGBoost model trained on 44,000+ real and fake news articles
- **Google Fact Check integration** — surfaces related claims and verdicts from verified fact-checkers
- **ClaimBuster integration** — scores how check-worthy a claim is
- **User feedback loop** — collects corrections via SQLite to support future retraining
- **Flask web interface** — clean, responsive UI for submitting and reviewing claims

---

## Model Performance

| Metric | Score |
|---|---|
| Accuracy | 93.89% |
| Precision | 91.80% |
| Recall | 97.94% |
| F1-Score | 94.77% |

Trained on the [ISOT Fake News Dataset](https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset) using TF-IDF vectorization with trigrams and an XGBoost classifier.

## How It Works

1. User submits a news claim via the web interface
2. Text is cleaned and stemmed using NLTK
3. TF-IDF vectorizer transforms the text into features
4. XGBoost model predicts **Genuine** or **Fake**
5. Google Fact Check API returns related verified claims
6. ClaimBuster scores how check-worthy the claim is
7. User can submit feedback to flag incorrect predictions

## Run Locally — Step by Step

### Prerequisites
Make sure you have the following installed before starting:
- **Python 3.9 or higher** — download from [https://www.python.org/downloads](https://www.python.org/downloads)
- **pip3** — comes bundled with Python
- **Git** — download from [https://git-scm.com](https://git-scm.com)

> On Mac, if you get `command not found: python`, use `python3` instead. On Windows, use `python` and `pip`.

---

**Step 1 — Clone the repo**
```bash
git clone https://github.com/ak2106-47/Fake-News-Detector.git
cd Fake-News-Detector
```

---

**Step 2 — Install dependencies**
```bash
pip3 install -r model/requirements.txt
```

> On Mac, if you get an error about `libomp`, run: `brew install libomp` then retry.

---

**Step 3 — Download the dataset**

The dataset is not included in this repo due to file size. You need to download it manually:

1. Go to [https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset](https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset)
2. Create a free Kaggle account if you don't have one
3. Click **Download**
4. Unzip the downloaded file
5. Place `Fake.csv` and `True.csv` into the `Datasets/` folder of this project

---

**Step 4 — Set up API keys**

The app uses two external APIs for fact-checking. It will still run without them — only the fact-check results will show errors, the ML prediction will still work.

If you want the full experience:
- **Google Fact Check API** — get a free key at [https://console.cloud.google.com](https://console.cloud.google.com) (enable the Fact Check Tools API)
- **ClaimBuster API** — get a free key at [https://idir.uta.edu/claimbuster](https://idir.uta.edu/claimbuster)

Then set up your `.env` file:
```bash
cp model/.env.example model/.env
```

Open `model/.env` and fill in your keys:
```
FACT_CHECK_API_KEY=your_google_fact_check_api_key_here
CLAIMBUSTER_API_KEY=your_claimbuster_api_key_here
```

---

**Step 5 — Build the model**

This trains the XGBoost classifier and saves `model.pkl` and `vectorizer.pkl` into the `model/` folder. This step takes **5–15 minutes** depending on your machine.

```bash
cd model
python3 build_model.py
```

You should see output like:
```
Loading datasets...
Processing text...
Fitting TF-IDF...
Training XGBoost...
Accuracy: 0.9389
✅ Saved vectorizer.pkl and model.pkl
```

---

**Step 6 — Start the app**
```bash
python3 app.py
```

---

**Step 7 — Open in your browser**

```
http://localhost:5000
```

Enter any news headline and click **Check** to get a prediction.

## Troubleshooting

| Error | Fix |
|---|---|
| `command not found: python` | Use `python3` instead |
| `ModuleNotFoundError: No module named 'flask'` | Run `pip3 install -r model/requirements.txt` |
| `libomp` error on Mac | Run `brew install libomp` |
| `_pickle.UnpicklingError` | Re-run `python3 build_model.py` — model files are missing or corrupt |
| API errors in the browser | Your API keys are missing or invalid — the ML prediction still works without them |

## Limitations & Future Work

- The current model uses TF-IDF + XGBoost, which is effective but does not understand context the way transformer models do. A future version could fine-tune a **BERT** or **RoBERTa** model for improved accuracy on short or ambiguous claims.
- The feedback collected in SQLite could be used to periodically retrain the model via `retrain_model.py`.
